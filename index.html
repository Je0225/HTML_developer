GERAÇÕES DE COMPUTADORES

(Voltar)

  • 1 Geração (1944-1959): marcada pelos computadores Harvard Mark 1(1944), Colossus (1946) e ENIAC (1946), os computadores eram gigantescos e ocupavam salas enormes, continham um grande número de Válvulas e Painéis,também faziam o uso de Plug's, nesta epoca surgiu o termo "bug".
  • 2 Geração (1959-1965): marcada pela subtituição das válvulas pelos transistores e sistemas batch.
  • 3 Geração (1965-1970): circuitos integrados,diminuição do tamanho dos computadores, com transistores, capacitores e multiprogramação.
  • 4 Geração (1971-atualmente):marcada pelo uso de personal computer, etc.
  • 5 Geração: Algumas pessoas dizem que já estamos nela, é marcada por cryptomoedas, metaverso, IA, computadores quanticos, etc.

SURGIMENTO DA WEB

(Voltar)
A internet é basicamente, um sistema de comunicação, que evolui de forma paralela aos computadores

O primeiro sistema de comunicação a surgir é o telégrafo, junto com o código morse na guerra fria (1950),e isso explodiu a tecnologia, neste cenário, também temos o ENIAC, que fazia calculo para o lançamento de mísseis

Ainda neste cenário de guerra fria, depois de muitas pesquisas e evoluções surge a ARPA ( atual ARPANET), com avanços em sistemas de comunicação em pacotes ( enviados um a um e que permitiam multiplos destinos) e estudos sobre nos, pontos de intersessão funcionando como check points (avõ dos roteadores)

A primeira palavra trafegada na internet foi 'LO', era pra ser LOGIN, mas a palavra não chegou ao outro ponto inteira devido a uma falha na conexão, que foi feita somente entre um ponto e outro. Com o passar do tempo, houve a criação do protocolo TCP IP, e foram surgindo mais pontos de conecção da ARPANET.

PROTOCOLO TCP/IP: Resumidamente, são conjuntos de protocolos que visam a conexão dos dispositivos, eles garantem que pacotes de informações cheguem a seu destino de maneira correta e segura, por meio disso toda a internet é endereçada, pois viabiliza troca de informações com as mais variadas origens e seus respectivos destinos.

WORLD WIDE WEB - WEB - WWW: È um sistema de documentos, que podem estar na forma de vídeos, sons, hipertextos, imagens, etc, que estão distribuidos na internet, em termos gerais, é a interface gráfica da internet, onde as informações podem ser facilmente distribuidas através de hipertextos, que são codificados em HTML, que é interpretado pelos navegadores e que passa a informação das páginas aos usuários. O WWW parece sinônimo de internet, mas em essência não é, pois a internet em si é a conexão que é feita pela rede de computadores, entre diversos servidores e clientes espalhados pelo mundo, e o WWW é o conteúdo e organização das informações trafegadas dentro dessa rede.

CLIENTES NA WEB

(Voltar)

CLIENTE:È quando o usuário consome a informação ( dados que serão solicitados ao servidor, e/ou modificados) provinda de um servidor conectado através da internet, um exemplo comum de clientes são os celurares e PC's convencionais, onde atravéz deles fazemos pesquisas na internet, usamos as redes sociais, jogamos online e etc, em todas essas ações, solicitamos dados de servidores para "carregar" as páginas e outros recursos que precisamos, por isso dizemos que a internet é uma rede mundial de computadores ( clientes e servidores conectados)

CLIENT-SIDE - LINGUAGEM DO CLIENTE - CLIENT SIDE SCRIPTING: São as linguagens executadas no lado do cliente, nas situações que a linguagem Server-side não alcança, e essa linguagen apenas o navegador vai entender

  • HTML (xHTML, HTML4, HTML5...), CSS (CSS2.1, CSS3...), JavaScript. Etc.

NAVEGADORES

(Voltar)

Os navegadores (browsers) são responsáveis por interpretar os arquivos do servidor (HTML, java, CSS, etc) inteles fazem isso de forma padronizada, mas as vezes pode acontecer de algumas informações se comportarem de maneiras diferentes de um navegador para o outro, esses problemas de responsividade e compatibilidade podem ser resolvidos com boas praticas e testes.

DESIGN RESPONSIVO OU RESPONSIVIDADE: Diferentes resoluções e distribuições de elementos na tela devem ser consideradas, pois usamos diferentes dispositivos para acessar as informações de um mesmo servidor

  • W3c:Onde resumidamente, os executivos de grandes corporações que mantem os navegadores, estipulam padrões a serem seguidos por todos os navegadores.
  • O primeiro navegador foi criado por Tim Berners-Lee, que inventou o WWW, inclusive, o nome deste navegador era WWW.

: Uma solução que é executada diretamente no browser, não sendo preciso realizar uma instalação na máquina do usuário. outra definição é: “tudo aquilo que é processado em um servidor terceiro”. As plataformas de e-commerce e as redes sociais são alguns dos exemplos que se enquadram nesse perfil, e o servidor pode ser apenas o servidor, ou ele pode também se conectar com o banco de dados, o lado do cliente não tem essa capacidade, na verdade, os navegadores possuem os bancos, mas é coisa mínima, e não é considerado um banco de dados robusto para aplicações web.

Quando um app é acessado pelo navgador, é uma aplicação web, mas quando esse mesmo app é instalado no dispositivo, não é mais uma aplicação web, e para aplicações instaladas, outras linguagens serão usadas.

SOBRE SERVIDORES

(Voltar)

Onde se hospedam os sites, servem as informações para os clientes,quando fazemos uma pesquisa, entramos na internet, "batenos" no servidor, e ele te retorna arquivos HTML, CSS< Javascript, imagens, audios...esses arquivos são chamados de arquivos estáticos(aruivos enviado do servidor para o cliente, são arquivos que são enviados ao navegador exatamente como estão no HD do servidor)e a expeiencia do usurário será afetada pela velocidade de conexão, e o navegador gera o que chamamos de cache, que são arquivos do servidor armazenados no pc do cliente, quando acessarmos novamente a mesma página, não se vai no servidor buscar de novo as informações que são estáticas ( que não mudam),e também, nos navegadores é implementado o refresh (atualize a pagina) ele vai novamente ao servidor e busca atuaizaçoes da página.

Servidores pra internet usam a conexão da internet, para fazer a comunicação entre eles. mas não significa dizer que ele só é acessivel pela internet, pois ele pode ser um servidor dentro de uma rede local, onde existe um computador central que ficam armazenados alguns tipos de arquivos, essa máquina é um servidor que serve de apoio para as outras máquinas da rede, isso é chamado de rede local, um exemplo de servidor domestico é quando temos somente um gabinete, onde os outros dispositivos mandam informações para ele.] È comum um mesmo servidor ser varios tipos de servidores ao mesmo tempo, mas não é muito víavel, pois quando solicitamos várias requisições, as outras podem falhar ou ficar lentas, pois a máquina precisa processar muitas coisas ao mesmo tempo, o ideal seria ter um servidor para cada coisa.

TIPOS DE SERVIDORES

  • DE ARQUIVOS:Um computador que centraliza as i.magens, audios, vídeos, etc, onde todos que estiverem conectados a tal rede conseguem acessar esses arquivos.
  • PROXY:: Está entre os clientes e outro servidor, por exemplo: uma empresa que quer bloquear o acesso a sites maliciosos, o proxy vai prover a internet aos funcionários com algumas regras, ele é um intermediario, realiza uma ponte entre a requisição do cliente com o servidor da web desejado. Sem ele, a solicitação é enviada diretamente ao servidor web, o que pode acarretar alguns perigos. Todavia, ele não tem apenas essa função, sendo utilizado também para manter a privacidade do usuário mascarando seu IP, além de armazenar dados, monitorar o acesso e demais funcionalidades. ou seja, antes de o funcionario "bater" na internet para entrar num site, o proxy irá dizer se ele pode entrar nesse site ou não, pode ser também usado para autenticação, ele analisa usuários e senhas e faz um proceso de validação, determinando o que cada um pode acessar ou ver.
  • FIREWALL:È considerado um servidor, mas com outro formato, forma uma barreira em redes de computadores para controlar o fluxo de dados entre um link da internet e a rede interna. Com isso, evitam-se roubos de informações empresariais e pessoais. Basicamente, filtra os dados que chegam na rede interna protegida por ele.
  • A maior diferença entre Proxy e Firewall: O frirewall impede o acesso a certos sites, o proxy não impede, mas somente filtra o que vem da internet, e fica como um intermediário entre o cliente e servidor web pra fins de anonimato, cache, filtro de navegação, e é importante ressaltar que esses dois podem estar na mesma máquina.
  • SERVIDOR DNS (DOMAIN NAME SERVICE):): È um servidor que detem uma lista de nomes e IPs (registro de todos os domínios da internet), não hospeda os sites, mas permite que as requisições cheguem aos lugares corretos, ele é basicamente, feito para a tradução de domínios para endereços de IP's.
  • FTP: É composto pelo conjunto de regras que diz para os computadores como transferir arquivos de forma simultânea entre diferentes sistemas online, esses arquivos não possuem nenhum tipo de criptografia, portanto não é tão seguro. O FTP pode ser usado dentro de uma rede interna de computadores, ou online entre diferentes servidores web.
  • INTRANET:Quando um site está disponível somente dentro de uma rede específica, essa rede corporativa se chama intranet, onde um servidor que ira servir as paginas para os clientes.
  • Existe o servidor, que quando acessamos a internet para entrar numa aplicação web, bate na web server/aplication server, e esse servidor tem o contato de outro servidor onde se encontra todo o banco de dados.

DIFERENÇAS DE HARDWARE E SOFTWARE EM SERVIDORES

(Voltar)

O servidor tende a ser um computador mais robusto, pois exige maior processamento e comunicação entre varios dispositivos, servidores podem ser simples como um gabinete, ou gigantescos que ocupam salas (data centers).

Os servidores utilizam SO's com recursos voltados para servidores ( windows server, ubuntu, centOS, Fedora, Debian, Oracle, Linux (* todos esses SO's citados são distribuições open source do Linux, exceto o win server) e servidores web (Apach, Nginx). Quanto aos servidoresde sites da web, existem os programas que dão apoio ao sistema operacional, quando ocorrer uma requisição ao servidor web, o sistema opreacional ira executar aquele programa específico, e esse programa irá fazer o processamento da informação/requisição.

SERVER-SIDE

(Voltar)

A linguagem server-side, basicamente, recebe o requerimento e faz o processamento. Depois, transforma o resultado em um XHTML e envia para o navegador, alguns exemploos de linguagens são: java, php, ruby, c#, js.

O QUE É LINGUAGEM DE PROGRAMAÇÃO?

(Voltar)

Linguagem escrita e formal que especifica um conjunto de instruções e regras usadas para gerar programas, *a linguagem de programação não vai resolver o problema, quem vai resolver o problema é o desenvolvedor usando as linguagens, vale ressaltar que cada linguagem se sai melhor em determinados aspectos.

LINGUAGEM DE ALTO NÍVEL:Estão mais próximas da linguagem humana do que a das máquinas (compostas por 0 e 1),feitas por pulsos elétricos, que somente cientistas entendiam.

HTML NÃO É UMA LINGUAGEM DE PROGRAMAÇÃO,pois não pode criar funcionalidades dinâmicas, HTML é basicamnete um arquivo de texto de marcação, que possui delimitadores que vão ser interpretados pelo navegador. cada um desses delimitadores possui um comportamento, ou seja, apenas estrutura um conjunto de seções, o mesmo se aplica para um arquivo CSV.
